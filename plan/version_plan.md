## 1. 版本号规则

| 版本号 | 含义 | 示例 |
|-------|------|------|
| v0.x.0 | 新增一个核心功能模块 | v0.2.0 — 新增主题/自动阅读/快捷键 |

- 旧代码不改，新功能写新模块

### v0.3.0 — 工具栏浮动优化（2026-06-26）

> 版本跳变：`v0.2.0` → `v0.3.0`

**工具栏浮动功能**（解决屏占比过高时原生工具栏被推出视口）：
- 动态触发阈值：屏占比 >85% **或** 元素实际溢出视口
- **顶部工具栏**：JS 计算 pixel 位置 → `position: fixed` 钉在视口顶部居中，鼠标移到顶部 56px 范围淡入
- **右侧功能按钮**：inline style 覆盖原生 `margin-left`，固定在视口右侧 20px，鼠标移入右侧 30px 感应区淡入
- 两个独立感应区 + 独立 CSS 类名（`wre-show-topbar` / `wre-show-controls`），互不干扰
- 屏占比 ≤85% 且无溢出时自动还原原生布局

**目录重构**：
- 按 `plan/` / `dev/` / `test/` / `release/` / `usage/` / `screenshots/` / `inbox/` 七大维度整理
- Edge 商店审核已提交

**里程碑**：阅读体验核心痛点（工具栏消失）彻底解决。

### v0.4.0 ～ v0.6.x — 主题稳定性修复系列（2026-06-27 ～ 2026-07-01）

> 版本区间：多轮小版本迭代，无独立大版本 Tag  
> 最终收敛版本：v0.7.0

**核心问题**：主题切换后正文文字颜色残留/不可见。

**探索过的路径**：
1. 多正文容器文字重刷（`.readerChapterContent` 全覆盖）
2. 官方主题按钮自动同步（程序化点击官方按钮 → 发现不生效 → 移除该方案）
3. `renderTargetContainer` display:none 强制恢复
4. `reader_float_*` 全屏浮层释放
5. 结束页分支（`need-pay-or-preview-branch`）文本信号识别与释放
6. Canvas 渲染时序诊断（多层 `0/30/100/300/800/2000ms` 延迟 paint）

**最终方案 — CSS `filter` 重构（v0.7.0 前）**：
- 放弃 JS 逐元素刷 inline style 的方案（根本问题是 canvas 渲染时序不可预测）
- 改为纯 CSS `filter` 控制：
  - 暗黑：`filter: invert(1) hue-rotate(180deg)`
  - 护眼：`filter: sepia(0.4)`
  - 明亮：`filter: none`
- 大幅简化 `applyThemeColors()`，移除 token 竞争、MutationObserver、6 级延迟 paint 等复杂逻辑

**里程碑**：A1-A4 四项单步主题测试、F1 快速连续切换测试全部通过。filter 方案从根本上消除了时序竞争。

### v0.8.1 — 滚动模式与工具栏体验优化（2026-07-07）

> manifest.json 版本号：`0.8.0` → `0.8.1`

**背景**：v0.8.0 追平上架后，针对**滚动阅读模式**与工具栏交互的一批体验优化 / Bug 修复补丁（不新增主线功能）。

**本版本改动**：
- 修复：屏占比 100% 时原生顶栏无法点击（顶部感应区遮挡）——顶栏浮现时抬升 z-index 至感应区之上
- 修复：滚动模式下屏占比未正确适配（正文溢出、不居中、水平滚动条）——滚动模式改作用于 `.app_content`，按视口百分比居中，不再套用翻页模式的固定 px 基准
- 修复：滚动模式屏占比刷新后丢失——就绪判定不再强依赖翻页模式容器 `.readerChapterContent_container`
- 新增：滚动模式工具栏浮动（正文变宽盖住工具栏时才自动浮动隐藏，感应区呼出）
- 新增：滚动模式自绘悬浮滚动条（隐藏原生、鼠标靠右淡入、可拖动跳转、随滚动实时更新）
- 优化：顶部/右侧感应区加大（顶 56→100px、右 30→120px），右侧感应区 z-index 提升，修复"难以触发"
- 优化：顶栏浮现时水平居中于视口
- ⏸️ 搁置：横屏/双栏快捷键、全屏加入主题面板（原因见第 3 节「还未确认的开发计划」）

**里程碑**：滚动阅读模式体验对齐翻页模式，工具栏交互更顺手。

### 还未确认的开发计划

1. 浏览器上架：将主流的浏览器都做，商店上架。
2. 增加用户反馈功能
3. 增加打赏功能：随喜赞叹，几个二维码
4. 文档需要全部优化整理。
5. 优化：滚动模式下，文字占比有问题
6. **横屏 / 双栏快捷键切换（⏸️ 搁置）**：原生按钮已定位为右侧工具栏 `.readerControls_item.isNormalReader`（第 4 个，tooltip「双栏阅读」）。但脚本模拟点击（`.click()` / 完整鼠标事件序列 pointerdown→mousedown→mouseup→click / React `onClick` 直呼）在翻页 + 滚动模式下**均无效**（class 不变、无浮层、正文无变化）。推测：按钮非 React 组件 + 微信读书校验 `event.isTrusted` 拒绝合成事件。**后续方向**：改用插件自绘 CSS 布局实现横屏/双栏，或深挖原生事件通道 / API。
7. **全屏模式加入主题面板支持切换（⏸️ 搁置）**：随第 6 项一并延后。
