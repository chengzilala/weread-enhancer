## 3. 详细功能需求

### 3.1 左侧悬浮入口

**需求描述**：
- 在页面左侧显示一个悬浮图标按钮（FAB）
- 点击图标展开/收起主菜单

**交互细节**：

- 点击页面其他位置 → 收起菜单
- 勿扰模式下：图标透明，鼠标移到左侧区域才显示

### 3.3 阅读设置面板

#### 3.3.1 屏占比调节
- **需求**：支持自由调节阅读区域的宽度
- **实现方式**：滑块控制，实时预览
- **范围**：50% - 100%
- **默认值**：微信读书默认宽度

#### 3.3.2 自动阅读
- **需求**：自动滚动页面，解放双手
- **参数**：
  - 滚动速度（慢速/中速/快速，或自定义数值）
  - 滚动方向（向上/向下）
  - 开始/暂停控制
- **交互**：点击开始按钮启动，再次点击暂停

#### 3.3.3 翻页设置
- **需求**：自定义滚轮翻页行为
- **参数**：
  - 滚轮灵敏度
  - 翻页动画开关
  - 每页滚动距离
#### 3.3.4 屏占比与目录、字号等功能同步

**背景**：当屏占比 ≥90% 时，微信读书原生的顶部工具栏（readerTopBar）和右侧功能按钮（readerControls，包含目录、字号、主题等）会被推出视口，导致无法操作。

**实现方案**：

- **触发条件**：双条件触发，任意满足即启用——
  - ① 屏占比 >85%（86% 及以上触发浮动，85% 刚好能放下工具栏保持原生）
  - ② 元素实际溢出视口（兜底，处理小屏、缩放等边缘情况）
- **顶部工具栏**：
  - JS 计算精确 pixel 位置，钉在视口顶部居中（position: fixed + transform: none），保持原始宽度不变形
  - 默认透明隐藏，鼠标移到浏览器顶部 56px 范围内淡入显示，移开后 0.4s 延迟淡出
- **右侧功能按钮**：
  - 通过 inline style（最高优先级）覆盖原生 margin-left，固定在视口右侧 20px 处
  - 保持 position: absolute 不变，确保字号/目录等弹窗菜单定位正确
  - 默认透明隐藏，鼠标移入右侧 30px 感应区淡入显示，移开后 0.4s 延迟淡出
- **互不干扰**：顶部和右侧各自独立感应区和 CSS 类名（wre-show-topbar / wre-show-controls），鼠标放哪边只显示哪边
- **退出条件**：屏占比 ≤85% 且无元素溢出时关闭浮动，还原原生布局

## 5. 技术实现方案

### 5.1 技术栈
- **Manifest Version**: V3
- **语言**: 原生 HTML/CSS/JavaScript
- **依赖**: 零外部依赖，纯原生实现

### 5.2 文件结构
```
微信读书插件/
├── manifest.json      # 插件配置
├── content.js         # 核心逻辑（UI 生成、事件处理）
├── content.css        # 样式文件
├── README.md          # 项目说明
├── Running.md         # 安装运行说明
└── RPD_需求文档.md    # 本文档
```

### 5.3 实现关键点

#### 5.3.1 Content Script 注入
- 在微信读书页面加载完成后注入
- 使用 `document_end` 时机
- 避免与页面原有代码冲突

#### 5.3.2 UI 隔离
- 使用 Shadow DOM 或独立的根元素
- CSS 类名加前缀（`wre-`）防止样式污染

#### 5.3.3 事件处理
- 防抖处理，防止重复触发
- 阻止事件冒泡，避免影响页面原有行为

## 8. 风险与注意事项

| 风险 | 应对措施 |
|-----|---------|
| 微信读书页面结构变化导致插件失效 | 选择稳定的 DOM 选择器，做好降级处理 |
| 快捷键与页面原有冲突 | 提供冲突检测，允许自定义快捷键 |
| 性能影响 | 优化代码，减少不必要的 DOM 操作 |
