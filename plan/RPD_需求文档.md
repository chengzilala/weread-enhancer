### 3.3 阅读设置面板

#### 3.3.1 屏占比调节
- **需求**：支持自由调节阅读区域的宽度
- **实现方式**：滑块控制，实时预览
- **范围**：50% - 100%
- **默认值**：微信读书默认宽度

#### 3.3.2 自动阅读
- **需求**：自动滚动页面，解放双手
- **参数**：
  - 滚动速度（慢速/中速/快速，或自定义数值）
  - 滚动方向（向上/向下）
  - 开始/暂停控制
- **交互**：点击开始按钮启动，再次点击暂停

#### 3.3.3 翻页设置
- **需求**：自定义滚轮翻页行为
- **参数**：
  - 滚轮灵敏度
  - 翻页动画开关
  - 每页滚动距离
#### 3.3.4 屏占比与目录、字号等功能同步

**背景**：当屏占比 ≥90% 时，微信读书原生的顶部工具栏（readerTopBar）和右侧功能按钮（readerControls，包含目录、字号、主题等）会被推出视口，导致无法操作。

**实现方案**：

- **触发条件**：双条件触发，任意满足即启用——
  - ① 屏占比 >85%（86% 及以上触发浮动，85% 刚好能放下工具栏保持原生）
  - ② 元素实际溢出视口（兜底，处理小屏、缩放等边缘情况）
- **顶部工具栏**：
  - JS 计算精确 pixel 位置，钉在视口顶部居中（position: fixed + transform: none），保持原始宽度不变形
  - 默认透明隐藏，鼠标移到浏览器顶部 56px 范围内淡入显示，移开后 0.4s 延迟淡出
- **右侧功能按钮**：
  - 通过 inline style（最高优先级）覆盖原生 margin-left，固定在视口右侧 20px 处
  - 保持 position: absolute 不变，确保字号/目录等弹窗菜单定位正确
  - 默认透明隐藏，鼠标移入右侧 30px 感应区淡入显示，移开后 0.4s 延迟淡出
- **互不干扰**：顶部和右侧各自独立感应区和 CSS 类名（wre-show-topbar / wre-show-controls），鼠标放哪边只显示哪边
- **退出条件**：屏占比 ≤85% 且无元素溢出时关闭浮动，还原原生布局

## 5. 技术实现方案

### 5.1 技术栈
- **Manifest Version**: V3
- **语言**: 原生 HTML/CSS/JavaScript
- **依赖**: 零外部依赖，纯原生实现

### 5.2 文件结构
```
微信读书插件/
├── manifest.json      # 插件配置
├── content.js         # 核心逻辑（UI 生成、事件处理）
├── content.css        # 样式文件
├── README.md          # 项目说明
├── Running.md         # 安装运行说明
└── RPD_需求文档.md    # 本文档
```

### 5.3 实现关键点

#### 5.3.1 Content Script 注入
- 在微信读书页面加载完成后注入
- 使用 `document_end` 时机
- 避免与页面原有代码冲突

#### 5.3.2 UI 隔离
- 使用 Shadow DOM 或独立的根元素
- CSS 类名加前缀（`wre-`）防止样式污染

#### 5.3.3 事件处理
- 防抖处理，防止重复触发
- 阻止事件冒泡，避免影响页面原有行为

### 9.1 阅读统计与数据导出（P0）

#### 9.1.1 阅读时长统计
- **需求**：自动记录用户在当前书籍的阅读时长
- **数据维度**：
  - 今日阅读时长（分钟）
  - 本周累计阅读时长
  - 本月累计阅读时长
  - 本书累计阅读时长
- **实现方式**：
  - 页面激活时开始计时，页面隐藏/失焦时暂停
  - 使用 `visibilitychange` 事件监听
  - 数据按书籍 URL 分组存储

#### 9.1.2 阅读进度追踪
- **需求**：记录每本书的阅读进度
- **数据维度**：
  - 当前章节 / 总章节数
  - 阅读百分比
  - 上次阅读时间
- **实现方式**：
  - 监听微信读书页面 URL 变化和章节切换
  - 定期（每 30 秒）保存一次进度快照

#### 9.1.3 书架概览面板
- **需求**：在主菜单新增「阅读统计」入口，展示统计数据面板
- **面板内容**：
  - 当前书籍阅读时长 / 进度
  - 最近阅读的书籍列表（最近 10 本）
  - 本周/本月阅读总时长汇总
- **交互**：点击书籍可跳转到对应书籍页面

#### 9.1.4 数据导出
- **需求**：支持导出阅读统计数据
- **导出格式**：JSON / CSV
- **导出内容**：全部历史阅读记录
- **触发方式**：在阅读统计面板中点击「导出」按钮

#### 9.1.5 存储数据结构（新增）
```javascript
{
  readingStats: {
    // key = 书籍 ID 或 URL hash
    "book_123": {
      bookTitle: "三体",
      bookAuthor: "刘慈欣",
      totalReadingTime: 3600,     // 秒
      lastChapter: "第12章",
      lastReadAt: "2026-07-07T10:30:00Z",
      progress: 45.2,             // 百分比
      sessions: [
        { date: "2026-07-07", duration: 1800, chapters: ["第10章", "第11章"] },
        { date: "2026-07-06", duration: 1200, chapters: ["第9章"] },
      ]
    }
  }
}
```

### 9.4 代码模块化重构（P1）

#### 9.4.1 当前问题
- `content.js` 单文件 **3200+ 行**，涵盖 UI 生成、主题管理、屏占比计算、自动阅读、快捷键、调试日志等所有逻辑
- 维护困难，新人难以理解代码结构
- 新增功能无处安放，只能在文件末尾堆砌

#### 9.4.2 重构目标
- **原则**：不重写旧代码，新增功能使用新模块；旧代码只在必要时提取
- **新文件结构**：
```
微信读书插件/
├── manifest.json
├── content.js              # 入口 + 初始化 + 旧逻辑（保持不动）
├── content.css             # 样式（保持不动）
├── modules/                # 新增模块目录
│   ├── stats.js            # 阅读统计与导出
│   ├── notes.js            # 笔记增强（划线复制、导出、批注聚合）
│   ├── themes.js           # 主题管理（预设、切换、自定义）
│   └── utils.js            # 通用工具函数（防抖、存储、DOM 操作等）
├── README.md
├── Running.md
└── plan/
    └── RPD_需求文档.md
```

#### 9.4.3 模块加载方式
- content.js 通过动态 `<script>` 标签注入模块文件
- 或通过 `chrome.runtime.getURL()` 加载模块
- manifest.json 中声明 `web_accessible_resources` 暴露 modules 目录

#### 9.4.4 重构分期
| 阶段 | 内容 | 说明 |
|-----|------|------|
| v0.9.0 | 新增 modules/ 目录 + utils.js | 新功能写入新模块，旧代码不动 |
| v1.0.0 | 逐步提取 themes.js | 把主题相关逻辑从 content.js 迁移到模块 |
